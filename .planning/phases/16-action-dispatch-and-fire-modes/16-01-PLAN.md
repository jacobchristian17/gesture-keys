<feature>
  <name>ActionResolver and ActionDispatcher</name>
  <files>gesture_keys/action.py, tests/test_action.py</files>
  <behavior>
    **ActionResolver (ACTN-01):**
    - resolve("fist") returns Action(key_string="space", fire_mode=FireMode.TAP, ...) when fist is mapped
    - resolve("unmapped_gesture") returns None
    - set_hand("Left") switches to left-hand action map; set_hand("Right") switches back
    - resolve_compound("open_palm", "swipe_left") returns Action for compound gesture mapping
    - resolve_compound with unmapped direction returns None

    **ActionDispatcher tap mode (ACTN-02):**
    - dispatch(FIRE signal for tap-mapped gesture) calls sender.send() exactly once
    - dispatch(FIRE signal for unmapped gesture) does nothing (no crash)

    **ActionDispatcher hold_key mode (ACTN-03):**
    - dispatch(HOLD_START signal for hold_key-mapped gesture) calls sender.press_and_hold()
    - dispatch(HOLD_END signal) calls sender.release_held() and clears _held_action
    - dispatch(HOLD_START for new gesture) after HOLD_START for old gesture: old keys released before new pressed (ordering verified)
    - dispatch(FIRE signal for hold_key-mapped gesture) calls sender.send() (tap behavior, not hold -- FIRE is always tap regardless of fire_mode)

    **ActionDispatcher compound fire:**
    - dispatch(COMPOUND_FIRE signal) resolves via resolve_compound and calls sender.send()
    - dispatch(COMPOUND_FIRE for unmapped compound) does nothing

    **Stuck key prevention (ACTN-04):**
    - release_all() calls sender.release_all() and clears _held_action
    - release_all() is idempotent (calling twice does not crash)
    - After release_all(), subsequent HOLD_END does nothing (no double-release)
  </behavior>
  <implementation>
    Create gesture_keys/action.py with:

    1. **FireMode(Enum)**: TAP = "tap", HOLD_KEY = "hold_key"

    2. **Action(frozen dataclass)**:
       - key_string: str
       - fire_mode: FireMode
       - gesture_name: str
       - modifiers: list (pre-parsed pynput Key objects)
       - key: object (pre-parsed pynput Key or str)

    3. **ActionResolver**:
       - __init__(right_actions: dict[str, Action], left_actions: dict[str, Action], right_compound: dict[tuple[str, str], Action], left_compound: dict[tuple[str, str], Action])
       - set_hand(handedness: str) -> None: switches _active_actions and _active_compound
       - resolve(gesture_name: str) -> Optional[Action]: pure lookup in _active_actions
       - resolve_compound(gesture_name: str, direction: str) -> Optional[Action]: lookup in _active_compound

    4. **ActionDispatcher**:
       - __init__(sender: KeystrokeSender, resolver: ActionResolver)
       - dispatch(signal: OrchestratorSignal) -> None: routes to _handle_fire, _handle_hold_start, _handle_hold_end, _handle_compound_fire
       - _handle_fire: resolve gesture, call sender.send() (always tap behavior regardless of fire_mode)
       - _handle_hold_start: resolve gesture, if fire_mode == HOLD_KEY call sender.press_and_hold(), set _held_action
       - _handle_hold_end: if _held_action, call sender.release_held(), clear _held_action
       - _handle_compound_fire: resolve_compound, call sender.send()
       - release_all() -> None: clear _held_action, call sender.release_all()

    **Key design decisions:**
    - FIRE signal always uses sender.send() regardless of fire_mode (FIRE = tap behavior from orchestrator)
    - HOLD_START only activates hold_key behavior if Action.fire_mode == HOLD_KEY
    - _held_action is None when no key is held; this is the single source of truth
    - Tests use Mock for KeystrokeSender to verify call sequences
  </implementation>
</feature>
