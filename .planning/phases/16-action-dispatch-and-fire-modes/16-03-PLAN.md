<objective>
Fix hold_key fire mode to produce continuous key output on Windows.

Purpose: UAT tests 2 and 3 failed because press_and_hold() sends a single SendInput key-down event, which Windows does NOT auto-repeat for programmatic input (only physical keyboard events get OS auto-repeat). The fix replaces the single key-down approach with an app-controlled tap-repeat loop managed by ActionDispatcher.

Output: ActionDispatcher gains a tick() method that sends repeated keystrokes at hold_repeat_interval (30ms/~33Hz) while a hold is active. Pipeline calls tick() every frame. Hold_key mode now produces visible continuous output.
</objective>
