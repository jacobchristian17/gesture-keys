  <behavior>
    DistanceFilter tests (tests/test_distance.py):
    - check() returns True when palm span (WRIST-to-MIDDLE_MCP Euclidean distance) >= min_hand_size
    - check() returns False when palm span < min_hand_size
    - check() always returns True when enabled=False, regardless of palm span
    - Transition from in-range to out-of-range logs DEBUG message exactly once (not every frame)
    - Transition from out-of-range to in-range logs DEBUG message exactly once
    - Repeated out-of-range frames produce no additional log messages
    - _compute_palm_span() returns correct Euclidean distance for known landmark positions
    - min_hand_size and enabled are settable via properties (for hot-reload)

    Config tests (tests/test_config.py -- new TestDistanceConfig class):
    - Config with `distance: {enabled: true, min_hand_size: 0.20}` loads distance_enabled=True, min_hand_size=0.20
    - Config with `distance: {enabled: false, min_hand_size: 0.20}` loads distance_enabled=False, min_hand_size=0.20 (value preserved)
    - Config without `distance:` section loads distance_enabled=False, min_hand_size=0.15 (defaults)
    - Default config.yaml (no distance section) loads with distance_enabled=False

    - Add fixture for landmarks with known WRIST(0) and MIDDLE_MCP(9) positions for computing expected palm span
