  <behavior>
    CameraCapture:
    - CameraCapture(camera_index=0) creates capture object
    - start() begins daemon thread, returns self
    - read() returns (ret: bool, frame: ndarray|None) with latest frame (no queue buildup)
    - read() returns (False, None) if no frame captured yet
    - stop() sets stopped flag, releases VideoCapture
    - Thread is daemon (dies with main process)
    - Frame access is thread-safe (uses lock)
    - read() returns a copy of the frame (not the mutable internal reference)
  </behavior>
  <action>
    1. Create `gesture_keys/detector.py` with `CameraCapture` class:
       - Follow RESEARCH.md Pattern 1 exactly (threading.Lock, daemon thread, latest-frame-only)

       - `start(self)`: spawn daemon thread running `_update`, return self
       - `_update(self)`: loop reading frames with lock until stopped
       - `read(self)`: acquire lock, return (ret, frame.copy() if frame is not None else None)
       - `stop(self)`: set stopped=True, release capture
       - Add error handling: if VideoCapture fails to open, raise RuntimeError with message "Camera index {N} could not be opened"
