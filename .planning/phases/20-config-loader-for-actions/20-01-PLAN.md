<feature>
  <name>ActionEntry dataclass and parse_actions()</name>
  <files>gesture_keys/config.py, tests/test_config.py</files>
  <behavior>
    ## ActionEntry dataclass
    - Frozen dataclass with fields: name (str), trigger (Trigger|SequenceTrigger), key (str), cooldown (Optional[float]), bypass_gate (bool), hand (str), threshold (Optional[float])
    - hand defaults to "both", bypass_gate defaults to False, cooldown defaults to None, threshold defaults to None

    ## parse_actions() function
    - Input: dict from YAML `actions:` section (action_name -> {trigger, key, ...})
    - Output: list[ActionEntry]
    - Calls parse_trigger() on each entry's trigger field
    - Infers fire_mode: :static -> tap, :holding -> hold_key, :moving -> tap, sequence -> tap (stored on ActionEntry or derivable from trigger state)

    ## Test cases:
    - parse_actions({"vol_up": {"trigger": "fist:static", "key": "up"}}) -> [ActionEntry(name="vol_up", trigger=Trigger(FIST, STATIC), key="up", ...)]
    - Entry with cooldown: 0.5 -> ActionEntry.cooldown == 0.5
    - Entry without cooldown -> ActionEntry.cooldown is None
    - Entry with bypass_gate: true -> ActionEntry.bypass_gate == True
    - Entry without bypass_gate -> ActionEntry.bypass_gate == False
    - Entry with hand: left -> ActionEntry.hand == "left"
    - Entry without hand -> ActionEntry.hand == "both"
    - Entry with threshold: 0.8 -> ActionEntry.threshold == 0.8
    - Entry with invalid trigger string -> TriggerParseError raised
    - Entry missing "trigger" field -> ValueError with clear message
    - Entry missing "key" field -> ValueError with clear message
    - Two entries with same trigger string -> ValueError("duplicate trigger")
    - hand: invalid_value -> ValueError
    - Moving trigger: {"trigger": "fist:moving:left", "key": "left"} -> ActionEntry with Trigger(FIST, MOVING, LEFT)
    - Sequence trigger: {"trigger": "fist > open_palm", "key": "enter"} -> ActionEntry with SequenceTrigger
    - Trigger uniqueness scoped per hand: same trigger with hand:left and hand:right is ALLOWED (different scopes)
    - Same trigger with hand:both and hand:left -> ValueError (both overlaps with left)
  </behavior>
  <implementation>
    1. Add ActionEntry frozen dataclass to config.py (above parse_actions)
    2. Add parse_actions(actions_dict: dict) -> list[ActionEntry] function
    3. parse_actions iterates entries, calls parse_trigger(), validates required fields, checks trigger uniqueness per hand scope, returns list
    4. Trigger uniqueness: maintain a dict of (trigger_string, hand_scope) -> action_name. "both" conflicts with "left" and "right"
  </implementation>
</feature>
