    MotionState dataclass (frozen=True):
    - `moving: bool`
    - `direction: Optional[Direction] = None`
    - Singleton `_NOT_MOVING = MotionState(moving=False)` to avoid per-frame allocation

    MotionDetector class following established detector pattern:
    - Constructor params: buffer_size=5, arm_threshold=0.25, disarm_threshold=0.15, axis_ratio=2.0, settling_frames=3
    - `WRIST = 0` constant (landmark index)
    - `update(landmarks, timestamp) -> MotionState` called every frame
    - `reset()` clears buffer and internal state
    - Property setters for all config params (hot-reload pattern from SwipeDetector)

    update() logic:
    1. If landmarks is None: set _hand_present=False, reset _moving/_direction, return _NOT_MOVING
    2. Hand entry detection: if not _hand_present, set _hand_present=True, _settling_remaining=settling_frames, clear buffer
    3. Settling guard: if _settling_remaining > 0, decrement, append to buffer but return _NOT_MOVING
    4. Append (x, y, timestamp) from landmarks[WRIST] to rolling deque buffer
    5. Need at least 2 entries to compute velocity; if not enough, return current state
    6. Compute velocity: displacement = sqrt(dx^2 + dy^2) between oldest and newest buffer entries, dt = time difference. Guard dt <= 0.
    7. Hysteresis transitions:
       - Not moving + velocity >= arm_threshold: classify direction, if valid -> moving=True, set direction, log transition
       - Moving + velocity < disarm_threshold: moving=False, direction=None, log transition
       - Moving + velocity >= disarm_threshold: update direction if changed (immediate flip, no pause required)
    8. Return MotionState(moving=self._moving, direction=self._direction) or _NOT_MOVING

    _classify_direction(dx, dy) -> Optional[Direction]:
    - Compute abs_dx, abs_dy. minor = min, major = max.
    - If minor > 0 and major/minor < axis_ratio: return None (too diagonal)
    - If abs_dx >= abs_dy: RIGHT if dx > 0 else LEFT
    - Else: UP if dy < 0 else DOWN (MediaPipe Y inverted)

    Logging: transition-only via logging.getLogger("gesture_keys"). Log motion armed/disarmed and direction changes.
