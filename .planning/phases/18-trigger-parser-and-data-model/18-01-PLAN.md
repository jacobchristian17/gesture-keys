    Create `gesture_keys/trigger.py` with:

    1. `TriggerState` enum: STATIC="static", HOLDING="holding", MOVING="moving"

    2. `Direction` enum: LEFT="left", RIGHT="right", UP="up", DOWN="down"

    3. `Trigger` frozen dataclass:
       - gesture: Gesture (from classifier.py)
       - state: TriggerState
       - direction: Optional[Direction] = None

    4. `SequenceTrigger` frozen dataclass:
       - first: Trigger
       - second: Trigger

    5. `TriggerParseError(ValueError)` with descriptive message

    6. `parse_trigger(trigger_string: str) -> Union[Trigger, SequenceTrigger]`:
       - Strip whitespace
       - If contains " > ", split on " > " into parts. If not exactly 2 non-empty parts, raise TriggerParseError. Parse each part as a single trigger (default state=STATIC if only gesture name given), return SequenceTrigger.
       - Otherwise parse as single trigger: split on ":"
         - 1 part: gesture only -- raise TriggerParseError ("state required, use gesture:static or gesture:holding")
         - 2 parts: gesture:state -- validate gesture against Gesture enum values, validate state against TriggerState enum values. If state is MOVING, raise TriggerParseError ("direction required for moving trigger, use gesture:moving:direction"). Return Trigger.
         - 3 parts: gesture:state:direction -- validate all three. If state is NOT MOVING, raise TriggerParseError ("direction only valid for moving triggers"). Validate direction against Direction enum values. Return Trigger.
         - 4+ parts: raise TriggerParseError

       Validation approach: Use try/except on enum construction or check `.value` in a set. Build VALID_GESTURES set from Gesture enum values, VALID_STATES from TriggerState values, VALID_DIRECTIONS from Direction values. Error messages must include the invalid token for debuggability.

    **EXCEPTION for sequence part parsing:** When parsing individual parts of a sequence (split on " > "), allow bare gesture names without a state (e.g., "fist" in "fist > open_palm"). Default to TriggerState.STATIC. This is handled by a private `_parse_single(part: str, allow_bare: bool = False)` helper.
