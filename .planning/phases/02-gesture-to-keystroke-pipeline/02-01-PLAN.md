  <behavior>
    Debounce (test_debounce.py):
    - IDLE + gesture -> ACTIVATING state, no fire
    - ACTIVATING + same gesture held for 0.4s -> FIRED, returns gesture
    - ACTIVATING + gesture switch -> resets timer, stays ACTIVATING
    - ACTIVATING + None -> returns to IDLE
    - FIRED -> immediately transitions to COOLDOWN on next update
    - COOLDOWN blocks all gestures for 0.8s
    - COOLDOWN + elapsed + gesture still held -> stays COOLDOWN (requires release)
    - COOLDOWN + elapsed + None -> returns to IDLE
    - Brief gesture (< 0.4s then None) never fires
    - State transitions logged at DEBUG level
    - Fire event returns the gesture (caller handles logging at INFO)

    Keystroke (test_keystroke.py):
    - parse_key_string("space") -> ([], Key.space)
    - parse_key_string("ctrl+z") -> ([Key.ctrl], "z")
    - parse_key_string("ctrl+shift+s") -> ([Key.ctrl, Key.shift], "s")
    - parse_key_string("enter") -> ([], Key.enter)
    - parse_key_string("a") -> ([], "a")
    - parse_key_string("CTRL+Z") -> normalizes to lowercase, same result
    - parse_key_string("unknown_key") -> raises ValueError
    - parse_key_string("ctrl+") -> raises ValueError (empty final key)
    - KeystrokeSender.send() presses modifiers in order, taps key, releases in reverse (test with mock Controller)
    - KeystrokeSender.send() releases modifiers even if key press raises exception (try/finally)
  </behavior>
