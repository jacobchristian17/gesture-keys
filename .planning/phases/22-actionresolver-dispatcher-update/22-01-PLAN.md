**ActionResolver changes (gesture_keys/action.py):**

Replace the constructor to accept 8 maps instead of 4:
- `right_static: dict[str, Action]` -- keyed by gesture value string (e.g., "fist")
- `left_static: dict[str, Action]` -- same
- `right_holding: dict[str, Action]` -- keyed by gesture value string
- `left_holding: dict[str, Action]` -- same
- `right_moving: dict[tuple[str, str], Action]` -- keyed by (gesture_value, direction_value)
- `left_moving: dict[tuple[str, str], Action]` -- same
- `right_sequence: dict[tuple[str, str], Action]` -- keyed by (first_gesture_value, second_gesture_value)
- `left_sequence: dict[tuple[str, str], Action]` -- same

Replace `resolve()` with `resolve_static(gesture_name: str)` and `resolve_holding(gesture_name: str)`.
Add `resolve_moving(gesture_name: str, direction: Direction) -> Optional[Action]` that looks up `(gesture_name, direction.value)`.
Add `resolve_sequence(first: Gesture, second: Gesture) -> Optional[Action]` that looks up `(first.value, second.value)`.
Remove `resolve_compound()` entirely.

Update `set_hand()` to switch all 4 active map pairs.

**ActionDispatcher changes (gesture_keys/action.py):**

Add `MOVING_FIRE` and `SEQUENCE_FIRE` branches in `dispatch()`:
- `MOVING_FIRE`: call `_handle_moving_fire(signal)` which calls `self._resolver.resolve_moving(signal.gesture.value, signal.direction)`. If action found, call `self._sender.send(action.modifiers, action.key)`.
- `SEQUENCE_FIRE`: call `_handle_sequence_fire(signal)` which calls `self._resolver.resolve_sequence(signal.gesture, signal.second_gesture)`. If action found, call `self._sender.send(action.modifiers, action.key)`.

Update `_handle_fire` to use `resolve_static` instead of `resolve`.
Update `_handle_hold_start` to use `resolve_holding` instead of `resolve`.

Update fixtures to provide 8 maps. Add new test classes:

- `TestResolveSequence`: resolve_sequence with mapped/unmapped gesture pairs
- `TestMovingFireDispatch`: MOVING_FIRE signal dispatches correctly, unmapped does nothing
- `TestSequenceFireDispatch`: SEQUENCE_FIRE signal dispatches correctly, unmapped does nothing
- Remove all `test_resolve_compound_*` tests
