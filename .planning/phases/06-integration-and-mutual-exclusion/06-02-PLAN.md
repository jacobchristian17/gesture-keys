    Modify BOTH __main__.py and tray.py detection loops identically. The changes are:

    **Change 1 -- Distance gating: add swipe_detector.reset()**

    In the distance gating block (where `if not in_range:` sets landmarks = None), add `swipe_detector.reset()` alongside the existing `smoother.reset()` and `debouncer.reset()` calls. This ensures stale swipe buffer data is cleared when the hand leaves range.

    In __main__.py this is around line 202. In tray.py around line 203.

    ```python
    if not in_range:
        if hand_was_in_range:
            smoother.reset()
            debouncer.reset()
            swipe_detector.reset()  # NEW: clear swipe buffer on distance transition
        hand_was_in_range = False
        landmarks = None
    ```

    **Change 2 -- Move swipe detection BEFORE static gesture classification**

    Currently the order is: static classify -> smooth -> debounce -> fire, THEN swipe detect -> fire.

    Reverse this: move the swipe detection block (the `if config.swipe_enabled:` block) to BEFORE the static gesture classification block.

    **Change 3 -- Add is_swiping suppression to static pipeline**

    Replace the current static gesture conditional:
    ```python
    if landmarks:
        raw_gesture = classifier.classify(landmarks)
        gesture = smoother.update(raw_gesture)
    else:
        gesture = smoother.update(None)
    ```

    With:
    ```python
    swiping = config.swipe_enabled and swipe_detector.is_swiping
    if landmarks and not swiping:
        raw_gesture = classifier.classify(landmarks)
        gesture = smoother.update(raw_gesture)
    else:
        gesture = smoother.update(None)
    ```

    When swiping is True, this feeds None to the smoother, which naturally decays any accumulated gesture state without a hard reset. The debouncer continues to receive the smoothed output (which will be None), so the activation timer resets naturally.
