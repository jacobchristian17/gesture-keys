    **Swipe detection in frame loop** (after the static gesture fire block, before config hot-reload):
    The swipe detector should receive landmarks BEFORE distance filtering sets them to None, but only when distance check passes (or distance is disabled). The current code sets `landmarks = None` when distance-filtered. SwipeDetector needs the original landmarks when in range.

    Add AFTER the debouncer fire block (after `logger.info("FIRED: %s -> %s", ...)`):
    ```python
    # Swipe detection (parallel path, bypasses smoother/debouncer)
    if config.swipe_enabled:
        swipe_result = swipe_detector.update(landmarks, current_time)
        if swipe_result is not None:
            swipe_name = swipe_result.value
            if swipe_name in swipe_key_mappings:
                modifiers, key, key_string = swipe_key_mappings[swipe_name]
                sender.send(modifiers, key)
                logger.info("SWIPE: %s -> %s", swipe_name, key_string)
    else:
        swipe_detector.update(None, current_time)  # Keep buffer clear when disabled
    ```

    NOTE: `landmarks` is already None when hand is lost or distance-filtered, so SwipeDetector.update(None, ...) will clear its buffer automatically. When landmarks exist and hand is in range, the swipe detector gets the raw landmarks (it only reads WRIST index 0). This satisfies SWIPE-04.
