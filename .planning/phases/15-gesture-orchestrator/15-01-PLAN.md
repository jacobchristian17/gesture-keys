  <behavior>
    ## Type Definitions (RED: test imports and construction)
    - OrchestratorAction enum with values: FIRE, HOLD_START, HOLD_END, COMPOUND_FIRE (same string values as DebounceAction)
    - LifecycleState enum: IDLE, ACTIVATING, SWIPE_WINDOW, ACTIVE, COOLDOWN
    - TemporalState enum: CONFIRMED, HOLD, SWIPING
    - OrchestratorSignal NamedTuple: action (OrchestratorAction), gesture (Gesture), direction (Optional[SwipeDirection])
    - OrchestratorResult dataclass: base_gesture, temporal_state, outer_state, signals list, suppress_standalone_swipe

    ## Constructor
    - Same config surface as GestureDebouncer: activation_delay, cooldown_duration, gesture_cooldowns, gesture_modes, hold_release_delay, swipe_gesture_directions, swipe_window
    - Starts in IDLE state with no base gesture

    ## Core State Transitions (port from test_debounce.py TestDebounceStateTransitions)
    - IDLE + gesture -> ACTIVATING (or SWIPE_WINDOW if gesture has swipe mappings)
    - IDLE + None -> stays IDLE
    - ACTIVATING + same gesture past delay -> ACTIVE(CONFIRMED) + FIRE signal
    - ACTIVATING + same gesture before delay -> stays ACTIVATING
    - ACTIVATING + None -> IDLE
    - ACTIVATING + different gesture -> restart ACTIVATING with new gesture
    - ACTIVE(CONFIRMED) + next frame (tap mode) -> COOLDOWN (1-frame ACTIVE for tap, like FIRED->COOLDOWN)
    - COOLDOWN + elapsed + None -> IDLE
    - COOLDOWN + different gesture -> ACTIVATING (or SWIPE_WINDOW) directly
    - COOLDOWN + same gesture -> stays COOLDOWN (blocked)

    ## Hold Mode (port from test_debounce.py TestHoldMode)
    - ACTIVATING + delay elapsed + hold mode -> ACTIVE(HOLD) + HOLD_START signal
    - ACTIVE(HOLD) + same gesture -> stays ACTIVE(HOLD)
    - ACTIVE(HOLD) + None -> start release delay timer
    - ACTIVE(HOLD) + None past release delay -> COOLDOWN + HOLD_END signal
    - ACTIVE(HOLD) + None within release delay + gesture returns -> stays ACTIVE(HOLD) (flicker absorption)
    - ACTIVE(HOLD) + different gesture -> ACTIVATING + HOLD_END signal

    ## SWIPE_WINDOW (port from test_debounce.py TestSwipeWindow)
    - IDLE + gesture with swipe mappings -> SWIPE_WINDOW
    - SWIPE_WINDOW + mapped swipe direction -> COOLDOWN + COMPOUND_FIRE signal
    - SWIPE_WINDOW + unmapped swipe direction -> stays SWIPE_WINDOW
    - SWIPE_WINDOW + window expired + gesture held -> ACTIVE(CONFIRMED) + FIRE signal
    - SWIPE_WINDOW + window expired + gesture lost -> IDLE

    ## Swiping Transitions (NEW -- absorbed from Pipeline)
    - Receives swiping flag each frame; detects entry (was not swiping, now swiping)
    - On swiping entry: resets internal state appropriately
    - On swiping exit (was swiping, now not): transitions to COOLDOWN with pre-swipe gesture suppression
    - No private state injection needed -- orchestrator handles internally

    ## Edge Cases (10 dedicated tests from CONTEXT.md)
    1. Direct gesture transitions: COOLDOWN + different gesture -> ACTIVATING/SWIPE_WINDOW
    2. Static-first priority gate: is_activating property returns True during ACTIVATING
    3. Swipe-exit reset: on swiping exit, orchestrator enters COOLDOWN with pre-swipe gesture
    4. Pre-swipe gesture suppression: after swipe exit, same gesture blocked by COOLDOWN
    5. flush_pending() fires pending gesture if in SWIPE_WINDOW with valid activating gesture
    6. Per-gesture cooldown durations from gesture_cooldowns dict
    7. Hold release delay grace period (flicker absorption in HOLD state)
    8. Compound swipe suppression: suppress_standalone_swipe=True during/after SWIPE_WINDOW
    9. Same-gesture cooldown blocking: COOLDOWN + same gesture -> stays COOLDOWN
    10. reset() method clears all state (used for hand switch)

    ## Properties
    - is_activating: True when outer_state is ACTIVATING
    - in_swipe_window: True when outer_state is SWIPE_WINDOW
    - activating_gesture: the gesture being activated or in swipe window
  </behavior>
  <implementation>
    Build orchestrator.py as a single module with:

    1. Type definitions at top: OrchestratorAction, LifecycleState, TemporalState, OrchestratorSignal, OrchestratorResult

    2. GestureOrchestrator class with handler-method dispatch pattern (same as GestureDebouncer):
       - __init__ with identical config params as GestureDebouncer
       - update(gesture, timestamp, *, swipe_direction=None, swiping=False) -> OrchestratorResult
       - reset() clears all state to IDLE
       - flush_pending() fires pending gesture if in SWIPE_WINDOW (for config reload)
       - Properties: is_activating, in_swipe_window, activating_gesture

    3. Outer FSM dispatch in update():
       - Track was_swiping internally for swiping entry/exit detection
       - Handle swiping transitions BEFORE outer state dispatch
       - Dispatch to _handle_idle, _handle_activating, _handle_swipe_window, _handle_active, _handle_cooldown

    4. Inner FSM within _handle_active():
       - Sub-dispatch based on temporal_state: CONFIRMED, HOLD, SWIPING
       - CONFIRMED (tap mode): auto-transition to COOLDOWN on same update (1-frame ACTIVE)
       - CONFIRMED (hold mode): transition to HOLD immediately (set by _handle_activating)
       - HOLD: same logic as GestureDebouncer._handle_holding() -- release delay, flicker absorption
       - SWIPING: stay in SWIPING while swiping flag is True (swiping exit handled before dispatch)

    5. Compound swipe suppression:
       - Track _suppress_until timestamp internally
       - When entering or in SWIPE_WINDOW, set _suppress_until = timestamp + swipe_window
       - Expose via OrchestratorResult.suppress_standalone_swipe boolean

    6. Key mapping from old states:
       - IDLE -> IDLE, ACTIVATING -> ACTIVATING, SWIPE_WINDOW -> SWIPE_WINDOW
       - FIRED -> ACTIVE(CONFIRMED) for 1 frame, COOLDOWN -> COOLDOWN
       - HOLDING -> ACTIVE(HOLD)

    Port ALL tests from test_debounce.py, adapting assertions:
    - DebounceState.FIRED assertions -> check outer_state=ACTIVE, temporal_state=CONFIRMED (or check signal emitted)

    - DebounceSignal assertions -> check OrchestratorSignal in result.signals list
    - Note: FIRED was 1-frame transient. In orchestrator, ACTIVE(CONFIRMED) for tap mode also auto-transitions to COOLDOWN. Tests checking state AFTER fire should check for COOLDOWN (since the next update transitions). Tests checking the FIRE frame itself should check for the signal in result.signals.
