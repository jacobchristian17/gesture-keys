### 1. App Starts with Actions-Only Config
expected: Run the app with `python -m gesture_keys --preview`. It should start without errors, show the camera preview window, and display the action count in the startup banner (e.g., "11 actions loaded"). No errors about missing gestures: or swipe: config sections.
result: pass

### 2. Static Gesture Fires Keystroke
expected: With preview running, make a gesture (e.g., fist). After the activation delay (~0.15s), the mapped keystroke should fire. The preview should show the gesture name and motion_state info (not "swiping: true/false").
result: pass

### 3. Motion Detection (Moving Gesture)
expected: Hold a gesture (e.g., open palm) and move your hand left/right/up/down. The MotionDetector should report motion direction. If you have a moving trigger configured (e.g., `open_palm:moving:left`), the mapped keystroke should fire while moving.
result: pass

### 4. Hold Gesture Fires Correctly
expected: Make a gesture and hold it still. If configured as a hold trigger (e.g., `fist:holding`), the held key should be pressed down while holding and released when the gesture changes. No stuck keys after releasing.
result: pass

### 5. Sequence Gesture (Two-Gesture Combo)
expected: If you have a sequence trigger configured (e.g., `fist>open_palm`), perform gesture A then quickly switch to gesture B within ~0.5s. The sequence keystroke should fire. Doing the gestures too slowly (>0.5s apart) should NOT fire the sequence.
result: pass

### 6. Config Hot-Reload
expected: While the app is running, edit config.yaml (change a key mapping in the actions: section). Save the file. The app should detect the change and reload without restart. The new mapping should take effect immediately.
result: pass

### 7. Activation Gate Still Works
expected: If activation_gate is configured, only the specified gesture(s) should arm the gate. Other gestures should be ignored until the gate is armed. Bypass gestures should fire regardless of gate state.
result: pass
