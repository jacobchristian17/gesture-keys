  <action>
    **Config changes (config.py + config.yaml):**
    1. Add `preferred_hand: str = "left"` to AppConfig dataclass.
    2. In `load_config()`, read `preferred_hand` from top-level config key: `preferred_hand = raw.get("preferred_hand", "left")`. Validate it's "left" or "right", raise ValueError otherwise.
    3. Add `preferred_hand: left` to config.yaml (commented out since it's the default, with a comment explaining it).

    **Detector changes (detector.py):**
    1. Change `HandDetector.__init__` to accept `preferred_hand: str = "left"`.
    2. Add instance state: `self._active_hand: Optional[str] = None` (tracks "Left" or "Right"), `self._prev_hands: set = set()` (handedness labels seen in previous frame).
    3. Change `detect()` return type from `list` to `tuple[list, Optional[str]]` — returns `(landmarks, handedness_label)` where handedness_label is "Left" or "Right" or None.
    4. Replace the right-hand-only filter with active hand selection logic:
       - Build a dict of detected hands: `{label: landmarks}` from result.handedness
       - If no hands detected: set `self._active_hand = None`, return `([], None)`
       - If exactly one hand detected: set `self._active_hand` to that hand, return its landmarks
       - If two hands detected:
         - If `self._active_hand` is not None and is in detected hands: return that hand (sticky)
         - If `self._active_hand` is None (startup): select `self._preferred_hand` (capitalize first letter to match MediaPipe label format e.g. "Left"), set as active, return it
         - If `self._active_hand` is set but NOT in detected hands (hand just disappeared while other is present): return `([], None)` and set `self._active_hand = None` — this implements "wait for clean single-hand detection". On the next frame if only one hand remains, the single-hand path will select it.
    5. Add a `reset()` method that clears `self._active_hand` to None (needed by Plan 02 for hand-switch state resets).

    **IMPORTANT NOTE on MediaPipe handedness:** MediaPipe labels are from the camera's perspective and are mirrored. The label "Left" in MediaPipe results actually means the user's left hand when the image is mirrored (default webcam). Verify this during implementation — the user wants "preferred_hand: left" to mean their left hand. MediaPipe's `category_name` already reflects the user's perspective (their left hand = "Left").

    **Test changes (test_detector.py):**

    - Add new tests for all behaviors listed above using the existing `_create_detector_with_mock()` and `_make_mock_result()` helpers.
    - The existing `test_left_hand_returns_empty` test should now expect left hand landmarks to be returned (since left hand is now supported).
    - The existing `test_both_hands_returns_right_only` should be updated to test sticky behavior.

    **Test changes (test_config.py):**

    - Add test that preferred_hand is parsed from YAML.
    - Add test that invalid preferred_hand value raises ValueError.
