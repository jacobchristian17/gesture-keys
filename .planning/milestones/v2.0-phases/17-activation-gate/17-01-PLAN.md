<feature>
  <name>Activation Gate Integration</name>
  <files>gesture_keys/config.py, gesture_keys/pipeline.py, tests/test_activation.py</files>
  <behavior>
    ## Config Schema (ACTV-02)
    - load_config() parses activation_gate section: enabled (bool, default false), gestures (list[str], default []), duration (float, default 3.0)
    - Missing activation_gate section returns defaults (enabled=false)
    - AppConfig has activation_gate_enabled, activation_gate_gestures, activation_gate_duration fields

    ## ActivationGate Basics (ACTV-01, baseline)
    - ActivationGate starts disarmed (is_armed() == False)
    - arm(t) makes is_armed() True
    - tick(t + duration) expires gate (is_armed() == False)
    - arm(t2) while armed resets timer (extends window)
    - reset() force-disarms

    ## Signal Filtering (ACTV-01, ACTV-03)
    - When gate is None (bypass mode): all signals pass through unchanged
    - When gate enabled and NOT armed: non-activation signals are suppressed (empty list)
    - When gate enabled and NOT armed: activation gesture signal arms gate and is consumed (not passed)
    - When gate enabled and armed: non-activation signals pass through
    - When gate enabled and armed: activation gesture signal re-arms and is consumed
    - ALL signal types (FIRE, HOLD_START, HOLD_END, COMPOUND_FIRE) for activation gestures are consumed
    - Multiple activation gestures work (either scout OR peace can arm)

    ## Gate Expiry Safety (ACTV-03)
    - Gate expiry calls dispatcher.release_all() (no stuck keys)
    - Gate expiry calls orchestrator.reset() (no stale HOLD state)

    ## Pipeline Integration (ACTV-01, ACTV-02, ACTV-03)
    - start() creates ActivationGate when config enabled and gestures non-empty
    - start() sets self._activation_gate = None when config disabled (bypass)
    - process_frame() ticks gate, detects expiry, filters signals
    - reload_config() updates or creates/destroys gate based on new config
    - FrameResult gains activation_armed: bool = False field

    ## Hot-Reload (ACTV-03)
    - Changing enabled from false->true creates gate
    - Changing enabled from true->false destroys gate and releases held keys
    - Changing duration updates gate.duration
    - Changing gestures updates _activation_gestures set
  </behavior>
  <implementation>
    **Config (config.py):**
    1. Add 3 fields to AppConfig: activation_gate_enabled (bool=False), activation_gate_gestures (list[str]=field(default_factory=list)), activation_gate_duration (float=3.0)
    2. In load_config(), parse activation_gate section from raw YAML following existing pattern (raw.get("activation_gate", {}))
    3. Add the 3 new kwargs to the AppConfig constructor call in load_config()

    **Pipeline (pipeline.py):**
    1. Import ActivationGate from gesture_keys.activation
    2. In __init__: add self._activation_gate = None, self._activation_gestures = set()
    3. In start(): if config.activation_gate_enabled and config.activation_gate_gestures, create ActivationGate(gesture=Gesture(config.activation_gate_gestures[0]), duration=config.activation_gate_duration) and set self._activation_gestures = set(config.activation_gate_gestures)
    4. Add _filter_signals_through_gate(signals, current_time) method per research pattern
    5. In process_frame() before signal dispatch loop: tick gate, detect expiry (was_armed -> not armed -> release_all + orchestrator.reset()), filter signals through gate
    6. In reload_config(): update or create/destroy gate based on new config activation_gate_* fields. If disabling, release_all(). If enabling, create new gate. If updating, adjust duration and gestures set.
    7. Add activation_armed: bool = False to FrameResult, set it from gate state in process_frame()
    8. In stop(): if gate was armed, release is already handled by dispatcher.release_all()

    **Key design decisions from research:**
    - ActivationGate stores single gesture (constructor requirement). Pipeline owns the set-based filtering via _activation_gestures.
    - signal.gesture.value (string) is compared against _activation_gestures set (strings) for matching.
    - Gate is None when bypass mode (not just disabled flag) for zero-overhead bypass.
  </implementation>
</feature>
